Overview

This is TARDis, which is a rapid scorm course authoring environment. You write courses without having to muck about with HTML or JavaScript or CSS (in fact, you can't). The courses are plain text in which commands telling the player how to render the page are dropped. There's a whole lot of commands which offer a lot of functionality.

This tool is written with the Chrome browser in mind, so use that. Most of it will work in Firefox, and it's never been tested in Internet Explorer and will never be. If you run into problems, there's a little orange bug on the black toolbar at the top which opens the helpdesk (Buggr). See bug reporting for more on how to use that.

Most bits of the screen will either have mouse-over text to describe what that button or area does, or have question mark icons to click. Most icons or colourful boxes are clickable.

Logging on

You need to log on to use this tool. You can edit your details by clicking on your name in the top right once you have logged on. The email address is used to send notifications from the bug reporter and todo list tools.

You will need a windows password to access the course share on the server (\\SERVER\Courses), which might be different to the password that you use to log onto the server. You shouldn't need to access the courses folder in most circumstances.

Logging on may also restrict the courses that you can see or edit, or limit certain functionality.

You can't log out directly; you have to close all browser windows and courses. You also have to do this if you get your password wrong!

Course listing page

The course listing page contains a list of the current courses filtered down to the settings you see in the box at the top. This filter remembers your selection forever (even after closing browsers), so you will probably only change it occasionally.

The buttons next to the filter box decide whether you want to match courses that match 'any' of the selected filter, or match 'all' of the selected options. Some combinations will return no results. The default is 'any'.

Mouse over most buttons or coloured areas for tool-tips to let you know what actions are.

Some of the links or buttons (like this one) will bring up menus or popup screens (like this overlay). You can generally close these either by clicking somewhere else, or using the little X icon to the top-right on windows.

Creating courses

Click the New Course button at the top. This makes a generic course using the CourseSuite layout template. You can also use the button splitter to bring down a special menu that lets you upload existing courses. The accepted format for these are either BMJ XML files, or LEO'S TOOL zip files.

Bug reporting

If you get an error, click on the Report a bug button. Take a screenshot of the error and paste it in where mentioned on that screen. You should get an email back about the ticket once it's been updated.

Todo lists

This is an idea storage area more than a bug logger. You can put notes about courses or about the system in general into this area, and tick them off at a later date.

Administrative functions

There are none.
